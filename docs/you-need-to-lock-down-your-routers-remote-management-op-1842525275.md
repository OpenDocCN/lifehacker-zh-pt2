# 您需要锁定路由器的远程管理选项

> 原文：<https://lifehacker.com/you-need-to-lock-down-your-routers-remote-management-op-1842525275>

我们以前说过，但我们要再说一遍:除非你正在使用网状路由器设置，这通常会将其配置与易于使用的应用程序和/或服务联系起来，否则你没有必要在路由器上启用 [远程管理](https://kb.netgear.com/976/Enabling-your-router-s-remote-management) 。这个设置在理论上听起来很棒——谁不想从远处访问他们路由器的配置呢？—但它实际上让你面临更多潜在的伤害，而不是好处。



老实说。大多数人一旦设置好路由器，就不会对它们做任何事情，除了咒骂它们，当他们的 wifi 感觉很慢时就重启它们。(你至少应该一个月左右*一次*一次一次[检查新的路由器固件](https://lifehacker.com/how-to-make-your-wifi-router-as-secure-as-possible-1827695547)一次，但那是以后的事了。)

我非常怀疑当你不在家时，你是否需要接入你的路由器设置，远程管理会给你的设备带来很多安全问题。考虑来自 [BitDefender](https://labs.bitdefender.com/2020/03/new-router-dns-hijacking-attacks-abuse-bitbucket-to-host-infostealer/) 的最新安全警告。据称，攻击者使用暴力技术猜测他们进入启用远程管理的不太安全的路由器的方式，以及具有任何种类的云配置选项的路由器。一旦他们做到了这一点，他们就会扰乱路由器的 DNS 设置，将用户的 web 请求重定向到恶意软件。

正如 BitDefender 所描述的:

> *有趣的是，通过改变路由器上的 DNS 设置，用户实际上会认为他们登陆了一个合法的网页，除非它是从不同的 IP 地址提供的。例如，当用户键入“example.com”时，网页不是由合法的 IP 地址提供，而是由恶意 DNS 设置解析的攻击者控制的 IP 提供。如果攻击者控制的网页是一个精确的传真，从浏览器地址栏的域名判断，用户实际上会认为他们登陆了一个合法的网页。*

重定向后，用户会看到一个弹出窗口，要求他们安装一个应用程序，以获得关于冠状病毒的新信息——你猜对了。一旦他们做到了这一点，他们最终会在系统上安装一个名为 Oski 的实用程序，该程序可以提取和发送浏览器凭据、存储的会话和保存的密码等数据。

避免这种恶意软件的最好方法是双重方法。首先，关闭路由器上的远程管理。它通常隐藏在某种高级设置菜单中，如果它存在的话，所以你必须做一些挖掘来找到它(或者查找你的路由器的具体说明)。如果你找不到它，但你知道你的路由器的配置与某种账户(例如 Linksys 的云服务)有关，那么你应该确保你有一个 [强密码](https://lifehacker.com/how-to-create-secure-passwords-that-arent-impossible-to-1825048324) 用于那个账户，而*不用于其他账户或服务*。

就这么简单。这完全说得通。您的路由器不应该默认使用简单登录，即使是基于 web 的 UI(如果适用)。首先，这意味着没有“管理员/密码”组合，如果你绝对*必须*给自己一个从世界任何地方回拨到你的路由器的机会，你需要用一个强密码把门关好并锁上。你会为你的笔记本电脑或智能手机做同样多的事情；为什么不是管理*你网络中一切的设备呢？*
