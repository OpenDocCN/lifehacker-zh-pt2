# i tried to block amazon from my life it was impossible 1830565336

> 原文：<https://gizmodo.com/i-tried-to-block-amazon-from-my-life-it-was-impossible-1830565336>

**第一周:亚马逊**

显然，我是个受虐狂。



我正在执行一项没有科技巨头 的 [生活任务——去发现这样的事情是否可能。不仅仅是通过纯粹的意志力，还通过技术手段，使用定制的工具，从字面上防止我的设备访问这些公司，以及他们访问我和我的数据。](https://specialprojectsdesk.com/preview/life-without-the-tech-giants-1830258056)

我从排除我认为最具挑战性的公司开始实验:万物商店。

像数百万其他美国人一样，我们在家里使用大量亚马逊产品。我们有一个 Echo，一个 Echo Dot，两个 Kindles，两张亚马逊 Prime Chase 信用卡，电视上的亚马逊 Prime 视频，以及两个 Prime 账户。(自我提醒:为什么我和我丈夫每人每年向亚马逊支付 119 美元？)

因此，可以说，亚马逊获得了我的大量资金和数据。我一个人平均每年在 Amazon.com 购物约 3000 美元。我已经成为一名忠实的购物者，以至于我几乎不知道还能去哪里网上购物。当我需要什么东西时，这是我第一个去的地方，任何东西——床单、尿布、卫生纸、万圣节服装、蓝牙耳机、给一个喜欢赌博的朋友的轮盘袖扣。基本上，每当我需要一个随机的实物时，我都会打开手机上的亚马逊应用。



是的，我操，我手机上有亚马逊的应用。我*那个*沉迷于这个公司。我并不孤单:据报道，亚马逊 [控制了 50%的在线商务](https://techcrunch.com/2018/07/13/amazons-share-of-the-us-e-commerce-market-is-now-49-or-5-of-all-retail-spend/) ，这意味着美国所有在线购物的一半，这是淫秽的。

亚马逊不仅仅是一个在线商店——这甚至不是我生活中最难割舍的东西。它的全球帝国还包括亚马逊网络服务(AWS)，这是一个庞大的服务器网络，为互联网的大部分提供了支柱，以及 Twitch.tv，这是在线游戏行业的支柱，以及全食超市，雅皮士饮食的有机支柱。

不让自己走进全食超市很容易，但我也想停止使用亚马逊的任何数字服务，从 Amazon.com(及其该死的应用程序)到任何其他使用 AWS 托管内容的网站或应用程序。为此，我寻求技术专家 Dhruv Mehrotra 的帮助，他为我建立了一个定制的 VPN，通过它来路由我的互联网请求。VPN 阻止任何进出亚马逊控制的 IP 地址的流量。我一直把我的电脑和手机连接到 VPN，还有我家里所有连接的设备；它应该清除亚马逊接触到的每一件数码物品。

不过，最终我们发现亚马逊太大了，难以征服。



AWS 是互联网最大的云提供商，去年创造了超过 170 亿美元的收入 。尽管亚马逊从其零售业务中赚取了更多的总销售额——超过 1000 亿美元——但如果你仔细查看其收益报告，你会发现**T5T7】其大部分利润 T9】都来自 AWS。宝贝，钱都在科技上。** 

AWS 于 2006 年推出，已经占领了互联网的大片领域。我的 VPN 最终阻止了亚马逊控制的 2300 多万个 IP 地址，导致了各种意想不到的伤亡，从主板和财富到美国政府问责局的网站。(政府机构喜欢 AWS，这可能就是为什么亚马逊，即将成为有三个“总部”的企业冥府守门狗，选择了位于弗吉尼亚州华盛顿郊区的阿灵顿作为其中之一。)我所依赖的许多智能手机应用程序也在封锁期间停止工作。

幸运的是，耶鲁大学法学院的网站可以运行，所以我可以下载反垄断专家莉娜·汗 2017 年的论文 ，该论文认为亚马逊是一个垄断企业，美国反垄断法目前的做法不适合监管——这是本周的必读书。

随着 VPN 的建立和运行，我开始想为什么这么多的网站还在工作。例如，Airbnb 的 [是 AWS 的著名用户](https://aws.amazon.com/solutions/case-studies/airbnb/?asc_campaign=InlineText&asc_refurl=https://gizmodo.com/i-tried-to-block-amazon-from-my-life-it-was-impossible-1830565336&asc_source=&tag=kinjagizmodolink-20) ，但我可以在那里搜索感恩节度假屋。我给 Airbnb 发了一封电子邮件，询问它是否仍在使用 AWS 进行托管，一位发言人证实了该公司的说法。(我也可以用 [这个很酷的工具](https://securitytrails.com/domain/airbnb.com/dns) 来证实，它告诉你一个网站的数字出处。)



这就是我和 Dhruv 如何发现我们拦网技术中的一个主要缺陷。事实证明，许多网站除了使用 AWS 这样的公司托管数字内容之外，还使用了一种叫做内容交付网络(CDN)的二级服务来加快网页加载速度。

互联网可能看起来像我们周围空气中看不见的水蒸气，但它也有一个重要的实体。AWS 在世界各地有庞大的服务器建筑，而 cdn 有大量较小的服务器。把 AWS 想象成一个站点数字包的中央仓库；cdn 是世界各地的店面，帮助人们更快地获得数据包，这样网站访问者就不必等待他们的数据从主仓库一路传来。

亚马逊运行自己的 CDN，名为 Cloudfront ，但它与其他公司如 Fastly、Cloudflare 和 Akamai 竞争激烈，Airbnb 似乎正在使用这些公司。

如果一个网站将 AWS 与非亚马逊 CDN 结合使用，我的拦截器会看到 CDN 使用的 IP 地址，并让 AWS 托管的内容通过。当我与 Gizmodo Media Group 的技术团队核实时，我发现我们自己的网站由 AWS 托管，并快速用作 CDN。就像 Airbnb 一样，Gizmodo 正在偷偷绕过我的拦截器。



尽管如此，我还是决心尽可能地屏蔽亚马逊。所以除了让 VPN 禁止亚马逊控制的所有 IP 地址，我还需要关闭我们家的亚马逊 Echo 和 Echo Dot。将它们连接到 VPN 是行不通的。我想干脆拔掉它们，但我担心有人会把它们插回去。(例如，我丈夫拒绝和我一起做街区工作，理由是他有一份“真正的工作”。)

"你为什么不把它们放在抽屉里？"Dhruv 问道。

令人难以置信的是，我没有想到这一点。回声已经成为家庭中的一个固定设备，我没有想到要把它扔掉。

这一周对我来说是一个持续的启示:亚马逊深深地嵌入了我的生活。不管我是否意识到，我每天都在重复使用它。没有它，我无法正常工作。



每当家里的纸巾用完时，我不得不跑到实体店，而不是打开我的亚马逊应用程序，这很烦人，但更大的挑战是失去我消费的几乎所有形式的数字娱乐。我最喜欢的浪费时间的 app，和朋友的话，加载不了。显然，我不能通过亚马逊视频观看节目，但我也不能观看网飞，因为尽管是亚马逊的竞争对手，网飞使用 AWS 服务其流媒体。HBO Go 是另一个受害者。

当我和丈夫去金门公园跑步时，我发现没有亚马逊的帮助，我无法在我的 Runkeeper 应用程序中记录跑步。没有 AWS，我也无法从图书馆下载一本有声读物到我的 Axis360 应用程序。Spotify 是最后一家存活下来的娱乐提供商(目前为 T0)，T1 是因为它的音乐存在于 T2 T3 谷歌云 T4 T5。感谢上帝。

在街区的第二天早上，我听到我女儿和我丈夫在客厅里尖叫着“阿蕾莎，阿蕾莎！”他们忘记了亚马逊回声的声音，Alexa 已经被驱逐出了房子。这种阻碍对我一岁的女儿 Ellev 来说尤其艰难，因为 Echo 是我们家唯一的音乐来源，也因为 Ellev 痴迷于三部电影( *Coco* 、 *Monsters Inc.* 和*The Incredibles)*，所有这些电影我们通常要么通过网飞观看，要么通过亚马逊购买视频观看。



Ellev 对我的实验很不高兴，特别是因为我长篇大论地解释她为什么不能听“E-I-E-I-O”或看“incweddies”对她来说毫无意义。一周的低谷是某天下午她为《超人特工队》哭了整整五分钟，尽管我最终用拼图分散了她的注意力。

除了娱乐选项变暗，我工作的基本工具变得无法使用，特别是加密的消息应用 Signal 和工作场所通信平台 Slack。

如果你不是一个在工作中使用 Slack 的人，就很难表达这有多么具有破坏性；它倾向于取代办公室会议、电子邮件和电话。没有 Slack，我基本上不知道整个亚马逊屏蔽周办公室都发生了什么，我的同事也不知道我在做什么。

这有一个心理上的好处。Slack 的目的是改善工作场所的沟通，但它也是一种工作场所监控的工具，这一点很明显，你名字旁边的绿点表明你是否准备好坐在键盘前，或者一个空的灰点表明你不在。通过屏蔽亚马逊，我不仅拆除了亚马逊对我生活的监视，也屏蔽了我同事的监视。



加密信息应用信号变暗是一件困难的事情，因为我越来越多地使用它来沟通，不仅仅是与有安全问题的人，还有我的丈夫和朋友。

我其实很惊讶 Signal 仍然使用 AWS，因为在 2018 年初，AWS 曾 [威胁要停止托管 Signal](https://gizmodo.com/amazon-bends-the-knee-to-autocrats-threatens-to-cut-of-1825697153) ，因为它正在伪装其互联网流量，以避免被专制政府关闭。最终，Signal 屈服于亚马逊的要求，因为正如 Signal 创始人莫邪·马林斯派克告诉我的那样，没有更好的选择。

AWS 的屏蔽还破坏了我女儿的日托所用来给我们发信息的两个应用程序。然而，一个技术上的怪癖使得日托所漏了一些东西；应用程序的通知来自苹果的服务器，而不是亚马逊的，所以我仍然可以看到更新(“Ellev 的新便盆”，“Ellev 的新饭菜”)，即使我不能检查应用程序，看看我的女儿在吃什么或排泄什么。

也许这样最好。我们随时随地获取任何信息的能力已经造成了一些不健康的数据成瘾。



没有亚马逊的一周肯定有积极的一面。我和丈夫打破了一天结束时看网飞电视节目的习惯，选择阅读或沉迷于克里比奇，这是一种我开始玩之前一直认为很无聊的纸牌游戏。此外，由于我们大多使用信号来相互发短信，我发现自己给他发的短信越来越少，而是和他谈论 IRL 的事情。

我们也让我们的女儿远离屏幕，这意味着和她一起玩或者带她去游乐场而不是给她一个“电影款待”我去户外跑步，而不是在跑步机上看网飞比赛。总的来说，访问互联网的部分越少，我就越少使用技术，这越来越成为我的人生目标。

但是去掉亚马逊也意味着严重限制我使用我们这个时代最重要的对话工具之一:链接语言的能力。



我问一个朋友我们在哪里见面吃饭，她发给我一个 Yelp 链接，我打不开。Dhruv 告诉我他正忙于“这个”，并发给我 [一个不工作的主板链接](https://motherboard.vice.com/en_us/article/gye79x/google-earth-stopped-mapping-this-military-installation) 。在最激烈的速记交流中，有人给我发了一个 Eventbrite 链接；分享的文字表明他们的伴侣已经在与癌症的斗争中死去，但我无法访问 Eventbrite 页面来确认。(我短暂地关闭了 VPN 来检查一下——感觉打破这个噱头还是值得的。)

我们用链接说话，即使是最具毁灭性的新闻，科技巨头也让自己成为链接翻译不可或缺的一部分。

Dhruv 会记录我的设备在一周内尝试连接亚马逊服务器的所有次数。这种情况发生了近 300，000 次，部分原因可能是因为应用程序无法从母舰处获得回复，导致反复发出 ping，直到我关闭它们。我的设备试图通过 3800 个不同的 IP 地址连接到亚马逊，这表明一周内有很多不同的应用程序和网站试图连接到亚马逊。

我在彻底禁止亚马逊上的失败并没有因为 CDN 的问题而停止。一天，我丈夫出去给我们买午饭，从全食超市带回来寿司。我吃了一块伊纳里，然后才想起我在吃亚马逊生产的食物。(我不愿意为了噱头而整肃。)



我无意中光顾亚马逊的另一次是当我意识到我们的汽车需要一个手机支架时，那是一种附着在通风口上的小塑料东西。我通常会在意识到我需要它的两分钟内，使用手机上的亚马逊应用程序，立即订购一个奇怪的小玩意，但不是这个星期。我最终从易贝订购了它。然而，当包裹到达时，它是一个黄色的信封，上面有顽强的“微笑”标志，旁边还有“亚马逊履行”的字样——即使是易贝的卖家也依赖它。

亚马逊已经彻底融入了现代生活的基础设施，融入了包括其竞争对手在内的许多公司的商业模式，这几乎是不可能避免的。

在她轰动一时的学术文章 中，现为美国联邦贸易委员会法律研究员的 Lina Khan 认为，亚马逊违反了反垄断法的精神，但监管机构未能采取行动，因为该法已经演变为忽略垄断，如果它们给消费者带来即时的低成本。



但可汗表示，我们在日常生活中对亚马逊的日益依赖带来了我们刚刚开始看到的危害，包括亚马逊能够剥削其员工(据报道，他们在瓶子里撒尿 ，以跟上公司的惩罚步伐)，能够对美国人的活动进行大规模数据挖掘(这意味着它可以根据对他们的了解对不同的人收取不同的价格， it 业 [过去曾尝试过这种方式](https://www.computerworld.com/article/2597093/retail-it/amazon-charging-different-prices-on-some-dvds.html) )，并能够击败那些原本会为消费者提供多种选择和价格的竞争对手(Diapers.com)。

亚马逊并不认为自己是垄断者。“水平宽度和垂直深度之间有一个重要的区别，”一位发言人在这篇报道发表后的一份声明中说。“我们经营多种业务，从零售和娱乐到消费电子和技术服务，我们在这些领域都有激烈和成熟的竞争。零售是我们最大的业务，我们在全球零售业中的份额不到 1%，在美国零售业中的份额约为 4%。”

但是，根据我本周的经历，我发现可汗的结论令人不寒而栗且颇有先见之明，尤其是她关于亚马逊吸引第三方卖家登陆其网站的观点。这使得卖家可以在短期内通过向亚马逊客户提供访问来赚取更多的钱，但亚马逊吸收了这些企业的数据，最终可以用更便宜的价格[击败他们](https://www.foxnews.com/tech/amazon-basics-is-killing-every-category-it-touches-including-batteries)。

鉴于亚马逊通过 AWS 托管网站获得了许多许多企业的数据，它可能正在大规模收集类似的竞争数据。事实上，在过去，它已经利用从 AWS 收集的见解对它认为在其云 中取得巨大增长的初创公司进行了 [投资。](http://www.reuters.com/article/amazon-cloud-idUSN1E7A727Q20111109)



“如果 AWS 产品经理不使用他们平台的使用模式数据来决定与谁以及如何开发有竞争力的产品，我会感到震惊，”马修·普林斯(Matthew Prince)说，他运营着 Cloudflare，这是一个内容交付网络，本周让我的屏蔽者感到沮丧。“他们已经在零售领域坚持不懈地这样做了，没有理由认为他们不会利用他们平台上的数据在数字服务领域做同样的事情。使用 AWS 的公司正在将关键的市场数据直接提供给这家公司，几乎可以肯定，这家公司有一天将成为他们最大的竞争对手。”

亚马逊没有回应关于它如何使用从托管其他公司网络服务的到收集的数据的询问。

最终，我认识到永久封杀亚马逊是不可能的。鉴于 cdn 的使用，这在技术上是不可能的，即使我们能找到一个完美的屏蔽，它也会将我与太多的关键服务和关键网站隔离开来，因为个人和职业原因，没有这些服务和网站我就无法工作。(老实说，我太喜欢看电视节目点播了。)我不能完全放弃亚马逊，但似乎还有其他人和公司应该非常努力地这样做。
