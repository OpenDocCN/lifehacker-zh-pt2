# 如果我使用双因素身份验证，但丢失了手机，会发生什么情况？

> 原文:[https://life hacker . com/what-do-I-do-if-I-use-two-factor-authentic ation-and-los-1668727532](https://lifehacker.com/what-do-i-do-if-i-use-two-factor-authentication-and-los-1668727532)

[双因素认证](https://lifehacker.com/no-one-knows-about-two-factor-authentication-and-privat-1838913065) 是一种 [必要的安全措施](https://lifehacker.com/two-factor-authentication-isnt-enough-to-keep-your-acco-1827867557) 它使用你的手机来帮助防止对你账户的未授权访问。如果你丢了手机，进入你的账户会变得更加困难，但这也是问题的关键。幸运的是，如果你找不到一个设备来验证你是不是真的你，你也不是没有选择。

Watch

双因素认证，就其本质而言，旨在防止在您无法使用手机(或其他认证设备)的情况下访问您的帐户。因此，没有多少方法可以在事后规避这一要求。然而，有许多方法可以防止这个问题发生。所以不要等到你丢了手机才设置它们。

(如果您当前被锁定，您可以直接跳到最后一部分。)

### 如果你有意扔掉手机...

如果你知道你要换手机，在你扔掉旧手机之前，确保*你换了一个不同的双因素认证设备(或者暂时什么都不用)。为了方便访问，这里有几个链接，如果您已经为一些常见服务启用了双因素设置(或了解如何这样做)，您可以在这些链接中更改双因素设置。请注意，这些链接可能只有在您登录到您的帐户时才有效。*

*   [谷歌](https://accounts.google.com/b/0/SmsAuthSettings#devices)
*   [收纳箱](https://www.dropbox.com/account/security)
*   [推特](https://twitter.com/settings/security)
*   脸书
*   [LastPass](https://lastpass.com/?ac=1) :在 web 上打开 LastPass，点击设置>多因子选项
*   [1 密码](https://my.1password.com/profile/2fa)
*   [不和](https://support.discordapp.com/hc/en-us/articles/219576828-Setting-up-Two-Factor-Authentication)
*   [抽动](https://www.twitch.tv/settings/security)

过程因服务而异，但基本原理是相同的。你将在你的新设备上安装一个应用程序，扫描条形码或从有问题的网站输入代码，并确认你拥有该设备。在大多数情况下，旧的授权码将停止工作，因此在更换之前请确保您已经确定。

如果你用短信，换手机应该没关系。只需激活您的新手机，代码就会出现在您的手机号码上。如果你使用认证器应用程序(我们推荐 Authy，我们稍后会谈到)，你可能会通过你的帐户设置交换你的认证器设备。

### **始终记下您的一次性备份代码**

我们怎么强调这一点都不为过。写下你的备份代码。如果您发现自己因为任何原因被锁定在帐户之外，包括您在放弃之前忘记禁用您的验证器(或者如果您的手机被盗，则无法禁用)，备份代码是重新访问您的帐户的最佳和最简单的方法。然后你可以设置一个*新的*认证器，很可能生成新的备份代码，并且像以前一样安全。

你可能听说过不应该写下你的密码，但是这些一次性密码是个例外。你一定要把它们打印出来或者写下来，放在你能找到的地方。理想情况下，它们应该与你的手机分开，或许放在一个防火的盒子里，或者与其他重要的纸质文件放在一起。不要只是将它们保存在笔记本电脑上的 Word 文档中，因为如果你的笔记本电脑坏了(或被偷了)，你就倒霉了。

与您的授权码不同，这些一次性代码不会改变。大多数网站也会告诉你它们什么时候被使用过，或者至少把它们从可用代码列表中标记出来。例如，谷歌提供了十个备份代码。当你使用一个时，代码列表从十个减少到九个(它们不会立即补充)，你会收到一封电子邮件，说代码已经被使用。这意味着，即使有人发现你的备份代码并使用它们来访问你的帐户，他们也很难不被发现。

### **使用第三方认证 app，如 Authy**

正如我们之前讨论过的，Authy 是一个很棒的应用程序，可以管理你在[iPhone](https://lifehacker.com/the-best-two-factor-authentication-app-for-iphone-1640526694)[Android](http://lifehacker.com/the-best-two-factor-authentication-app-for-android-1638791349)，甚至 [你的电脑](http://lifehacker.com/authy-brings-two-factor-authentication-to-your-pc-no-n-1569284437) 上的双因素账户。这不仅为您提供了一个“备份”设备以防您丢失手机，因为您的令牌 [会在您的各种设备](https://authy.com/features/multiple-devices/) 之间同步，而且还可以非常轻松地将您的令牌从一个设备迁移到另一个设备(比如，如果您有了一部新手机)。只需同步新设备并取消旧设备的授权。

为了在您的设备上设置同步令牌，您需要首先将 Authy 设置为您的主要双因素身份验证应用。如果你目前使用谷歌认证器或其他应用程序来获取你的代码，你需要检查你的账户并设置 Authy，很可能使用你必须扫描的二维码，就好像你正在切换到一个全新的设备。然后，按照以下步骤将 Authy 同步到第二个设备:

1.  打开主设备上 Authy 中的设置，然后轻按“设备”。
2.  启用“允许多设备”
3.  在您的第二台设备上，安装 Authy。
4.  当你第一次打开应用程序时，它会提示你输入电话号码。输入主设备的电话号码。
5.  在弹出窗口中，点击“使用现有设备”
6.  在您的主设备上，您将收到一条通知，要求您确认是否添加了新设备。点击“接受”
7.  在提示您确认批准该决定的框中键入“OK”。
8.  返回主设备上的设置，再次点击“设备”。
9.  停用“允许多设备”这可防止添加任何额外的设备，同时现有的已连接设备保持活动状态。

为所有连接到 Authy 的设备启用 PIN 码(或指纹/面部锁)也是一个好主意。(您需要在我的帐户>安全性中为每台设备单独执行此操作)。这样，即使有人能够接触到你的设备，他们也很难看到你的代码。

对于那些担心这种方法安全性的人来说:你所有的认证令牌都是在本地加密的(使用复杂的密码，而不是保护应用程序本身的四位数 PIN)，所以 Authy 的服务器和沿途任何窥探的第三方都不能访问令牌。

### **获得备用手机短信验证码**

虽然一些身份验证方法需要应用程序，但几乎所有方法都至少提供了使用 SMS 代码作为备份选项。它没有 [作为专用验证器应用程序或硬件令牌的解决方案](https://lifehacker.com/two-factor-authentication-isnt-enough-to-keep-your-acco-1827867557) 安全，但如果你丢失了手机，获得一个备份设备并通过运营商激活它，将允许你向与你的帐户关联的电话号码发送短信。

坏处呢？短信很多 [更有可能被黑客攻击](https://www.theverge.com/2017/9/18/16328172/sms-two-factor-authentication-hack-password-bitcoin) 甚至*如果*有人无法访问你的设备，包括令人畏惧的 [sim-swap 攻击](https://lifehacker.com/how-to-prevent-and-respond-to-a-sim-swap-scam-1835627474) 。当然，攻击者也需要您的密码才能使用特定的帐户进行任何操作，但是基于文本的身份验证仍然是一种不如双因素身份验证安全的方法，因为这需要他们实际接触您的身份验证设备才能侵入您的帐户。

### **如果你被锁在门外(还没有准备好)该怎么办**

虽然你有几种方法做最坏的打算，但事情还是发生了。你的手机掉进了井里，你丢失了带有备份代码的便利贴，而今天恰好是你的谷歌账户要求你重新验证的日子。运气不好。

有时，您仍可以给提供您想要访问的服务的公司打电话或发信息。坏消息是，如果公司能够做到的话，账户恢复过程通常需要几个工作日才能完成。其他公司(如 [Discord](https://support.discordapp.com/hc/en-us/articles/115001221072-Lost-Two-Factor-Codes) )会告诉你，如果备份选项失败，他们将无法为你提供访问你的账户的权限。你必须开一个全新的账户，这不是一个理想的解决方案。

这就是掌握备份选项非常重要的原因。然而，在最坏的情况发生时，这里有一些链接，告诉你如何(或是否)可以重新访问你的帐户以获得各种服务:

*   [谷歌](https://support.google.com/accounts/answer/185834?hl=en#phone)
*   [收纳箱](https://www.dropbox.com/en/help/364)
*   [推特](https://support.twitter.com/articles/20170409-i-m-having-trouble-with-login-verification)
*   脸书
*   [LastPass](https://lastpass.com/support.php?cmd=showfaq&id=2613)
*   [1 密码](https://support.1password.com/two-factor-authentication/)

像往常一样，一点点预防措施是值得的，在您丢失验证器设备后，尝试重新访问所有重要帐户的混乱和心痛是值得的。
