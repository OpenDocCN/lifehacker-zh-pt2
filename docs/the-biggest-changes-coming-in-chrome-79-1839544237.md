# Chrome 79 带来的最大变化

> 原文：<https://lifehacker.com/the-biggest-changes-coming-in-chrome-79-1839544237>

谷歌 Chrome 的下一个重大更新 Chrome 79 已经正式进入测试阶段，这意味着官方公开发布将在未来几周内到来——尽管我们大多数人还没有适应 Chrome 78 最近推出的。一旦更新推出，用户可以期待一些很酷的新功能，包括在多个同步设备上共享文本和链接，新的虚拟现实支持，更新的安全性，甚至是一个新的工具，可以抑制 Chrome 对系统内存的贪婪需求。



如果你想早点试用 Chrome 79，你可以 [下载并安装测试版](https://www.chromestatus.com/features/schedule) ，但对于那些宁愿等待任何挥之不去的错误被压扁的人来说，这里有一个 Chrome 下一次重大更新的最大功能的预览。

### 新功能

*   **标签冻结:**桌面用户很清楚[Chrome 能吃掉多少内存](https://lifehacker.com/why-chrome-uses-so-much-freaking-ram-1702537477) ，但 Chrome 79 将增加一个新的“标签冻结”功能，防止打开的标签在后台超过五分钟后执行新任务(不过这不会影响标签在后台播放媒体)。Chrome 的标签冻结功能将在 Windows、Mac、Linux 和 Chrome OS 上可用。
*   **跨平台剪贴板:**同时登录 Chrome 桌面和 Chrome Android 浏览器的用户将能够使用新的端到端加密剪贴板菜单在两个设备之间同步和共享文本。该功能将要求用户启用 Chrome sync。

### 新的虚拟现实开发工具

Chrome 79 官方公告中的头条功能是一个新的虚拟现实 API，开发者可以使用它来为智能手机和虚拟现实耳机创建虚拟现实和基于 AR 的网络应用 。

普通 Chrome 用户实际上能看到多少这一新功能取决于有多少网站、应用和游戏是用它开发的。也就是说，其他浏览器，如 Firefox Reality，Oculus Browser，Microsoft Edge 和 Helio 也都支持新的 API，所以我们肯定有可能在不久的将来看到更多具有 VR 或 AR 功能的网站和应用程序。

### 安全更新

*   **来自 HTTPS 域名系统的更多隐私:** Chrome 79 将最终为一小部分用户测试浏览器的 HTTPS 域名系统(DoH)。DoH 是一种更安全的访问网站的方法，因为它保护了浏览器用户的身份(你可以在这里 了解更多关于它的信息)。DoH 最初是为 Chrome 78 宣布的，但被推迟到 Chrome 79。
*   **更新了网站的安全要求:**用户可能会注意到，一些以前被认为是安全的网站在 Chrome 79 中会显示为“不安全”。这是因为 Chrome 不再支持某些安全协议，以努力让网站采用更好的安全标准。

### 其他变化

除了这些大的更新，Chrome 79 还有一大堆更多的技术变化和更小的调整。大多数这些对普通用户来说并不太重要，但对于那些好奇的人来说，它们被列在了 Chrome 开发者页面上。
