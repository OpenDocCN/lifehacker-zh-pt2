# 立即更新 Windows 10 以阻止“SMBGhost”

> 原文：<https://lifehacker.com/update-windows-10-now-to-block-smbghost-1843968661>

当连国土安全部的网络安全和基础设施安全局都开始对你未打补丁的 Windows 10 系统感到紧张时，也许是时候确保你已经从 Windows Update 下载了你需要的一切。



这一次，该机构正在对出现的新的概念验证攻击做出反应，这些攻击与 3 月份发现的一个漏洞有关——是的，三个月前。这个名为“SMBGhost”的漏洞利用了 Windows 的服务器消息块协议的一个问题，该协议可以让攻击者无限制地在受影响的机器上运行他们想要的任何东西。(显然，这包括服务器，但也包括连接到已被攻击的服务器的任何未打补丁的客户端。)

这还不是全部，正如 TechCrunch 的 [扎克·惠特克](https://techcrunch.com/2020/06/08/homeland-security-windows-10-wormable-bug/) 所描述的:

> *更糟糕的是，因为代码是“可蠕虫化的”，它可以跨网络传播，类似于*[*【NotPetya】*](https://techcrunch.com/2018/02/15/uk-accuses-russia-of-2017s-notpetya-ransomware-attacks/)*和* [*WannaCry 勒索软件攻击*](https://techcrunch.com/2019/07/08/the-wannacry-sinkhole/) *如何在全球传播，造成数十亿美元的损失。*
> 
> 尽管微软几个月前发布了一个补丁，但成千上万台面向互联网的计算机仍然容易受到攻击，这引发了该建议。

为了确保安全，您需要做的就是确保您已经安装了 Windows 10 的最新更新。就是这样。在你的家用机器上做到这一点非常容易——事实上，如果你经常使用它们并把它们连接到互联网上，它们应该已经更新了。

不过，奇怪的是。如果你使用的是比 1903 年版本(去年 5 月发布)更老的 Windows 10 版本，你就没问题了。您的操作系统还不支持 SMBv3.1.1 压缩，这是 SMBGhost 正在利用的 bug 的来源。

因此，从某种奇怪的角度来看，*而不是*更新比安装一个主要更新并对其余部分漠不关心更能让你免受这种攻击。然而你不应该继续这种做法。截至我们撰写本文时，是时候更新到 Wwindows 的最新版本了—[2004](https://lifehacker.com/how-to-install-microsofts-big-windows-10-may-update-rig-1842925015)版本—并确保您保持在您的补丁星期二更新和任何其他关键的计划外更新的顶部。

但是也有一个警告。你肯定知道，微软的各种 Windows 10 更新往往会出现一些问题。以至于可能*不值得你在每一个更新发布后马上安装它。正如伍迪·莱昂哈德为 [电脑世界](https://www.computerworld.com/article/3561576/microsoft-patch-tuesday-is-nigh-pause-updates-now.html) 所写的:*

> **...我们确实偶尔会遇到需要立即关注的紧急补丁，但它们极其罕见，而且总是广为人知——通常在发布后几小时内。我们看到永恒之蓝、Sasser 和一些鲜为人知的安全漏洞。即使在这些情况下，白痴们也需要几周或几个月的时间才能将一个已知的漏洞变成主流攻击。**
> 
> *相比之下，每个月我们都会看到补丁的问题。锁定系统。丢失的文件。混乱的应用程序。未记录和未宣布的更新。如果你不太熟悉 Windows 补丁的麻烦——并且确信你真的不应该在微软补丁一发布就让你的机器接触它们——看看我的 [*补丁警报*](https://www.computerworld.com/article/3216425/microsoft-patch-alert-may-2020.html) *系列中每月提交的三年的问题报告。**

*如果我是你——我也是这么做的——我会确保我使用的至少是 Windows 版本。然后我会使用它暂停 Windows 更新的功能，通过**设置>更新&安全**、**T3】来阻止你的操作系统在更新发布后立即下载和安装。***

*至于你需要等多久才能安装一个，这取决于你自己和更新的严重程度。如果一个更新正在修补一个零天的漏洞，你可能想更快地安装它；如果这是一个巨大的功能更新，你可能会等待一周(或两周)以确保破坏系统的错误没有在更新的公开发布中暴露出来。*

*这是征税吗？是的。你会忘记这件事吗？当然可以。当你不明白为什么你的系统在星期二工作得很好，但是在星期三早上却咳嗽出一些可怕的故障时，你还会记得吗？你现在会的。*

*这就把我们带回了 SMBGhost。至少，如果你使用的是 1903 或更高版本，请确保你已经用 KB4551762 更新了 Windows 10。你可以通过在**设置>更新&安全>查看更新历史**查看你的更新历史来查看你是否已经有了。也有可能更新没有出现——我的系统就是这种情况，因为我现在运行的是 Windows 10 version 2004。*
