# 要在公共 Wifi 上保持安全，你需要的不只是 HTTPS

> 原文:[https://life hacker . com/you-need-than-than-https-to-stay-safe-on-public-wifi-1841831844](https://lifehacker.com/you-need-more-than-https-to-stay-safe-on-public-wifi-1841831844)

你不应该再害怕使用公共 wifi 了。或者，至少现在你不用那么害怕了。虽然这并不意味着你应该盲目地连接到“免费互联网”、“xfinity”或其他随机、开放的热点，而没有一点常识——和一些软件备份——我们已经从人们使用 Firesheep 等软件工具来嗅你在公共 wifi 上做的一切的时代走了很长一段路。

Watch

至少，这是电子前沿联盟的立场。该组织最近 [发表了一篇文章](https://www.eff.org/deeplinks/2020/01/why-public-wi-fi-lot-safer-you-think) 声称你对使用公共 wifi 的任何恐惧都有点没有根据:*...由于 HTTPS 加密在大多数流行网站上的广泛部署，避免公共 Wi-Fi 的建议大多已经过时，适用于比以前少得多的人。”*

毫无疑问，如今我们生活在一个 HTTPS 世界。根据谷歌的统计，美国 Chrome 用户的大部分联系都是通过这种安全的形式 HTTP 进行的(93%)，而这个数字 [在全球范围内还在增长](https://transparencyreport.google.com/https/overview) 。类似地，网络前 100 个站点中的 96 个(不包括谷歌)目前默认使用 HTTPS，并且所有的*都支持* HTTPS 连接——这就是为什么你绝对应该在你的浏览器中使用类似[**HTTPS Everywhere**](https://www.eff.org/https-everywhere)的插件，以便在任何可能的时候强制使用这些更安全的连接。

当你这样做时，EFF 指出:

> *...从您的 ISP 到互联网骨干提供商，再到网站的托管提供商，通信路径上的任何人都可以看到他们的域名(例如 wikipedia.org ),当您访问他们时。但这些人看不到你在这些网站上访问的页面(如 Wikipedia . org/contential-topic)、你的登录名或你发送的消息。他们可以看到您访问的页面大小以及您下载或上传的文件大小。当您使用公共 Wi-Fi 网络时，其覆盖范围内的人可以选择收听。他们可以看到元数据，就像你在家浏览时，你的 ISP 可以看到的那样。”*

这一切都很好，但我不会盲目地连接到你看到的每个公共 wifi 热点，假设你完全没有障碍。首先，对于某人来说，建立一个虚拟的公共 wifi 并记录你正在使用的协议和端口 [并不难，就像这个](https://hackernoon.com/a-hacker-intercepted-your-wifi-traffic-stole-your-contacts-passwords-financial-data-heres-how-4fc0df9ff152) 。网络钓鱼登录和密码，尤其是如果一个人没有注意浏览器地址栏中显示的内容，同样是微不足道的。那些完全不懂技术的人甚至可能被说服安装一个流氓应用程序来实现“免费上网”哦。

并且记住，ZDNet 的 [大卫·格威兹](https://www.zdnet.com/article/reader-question-answered-if-i-have-https-do-i-need-a-vpn/)的中写道:

> “关于 https 加密需要考虑的另一件事是，它只对你的网络流量进行加密。https 协议不涉及任何其他互联网活动，因此需要自己的加密。其他活动的例子包括基于网络的视频游戏，它可能会以明文形式发送您的帐户、密码，甚至信用卡信息；电子邮件程序；或者甚至是本地运行的会计程序。”

如果真的想在公共 wifi 上保持安全，需要多管齐下。

#### 你真的需要无线网络吗，或者你可以用无线网络连接代替 T2？

考虑一下你是否真的需要无线接入，以及这个网络是否是你可以信任的(机场指示你应该使用的官方 wifi 网络)，或者是你觉得看起来很诱人且不贵的(一个随机的“免费 WIFI HERE”SSID)。虽然如果你连接到星巴克的公共 wifi，你还没有脱离险境，但实际上并不是这样，你至少比连接到一个明显的陷阱稍微安全一些。

#### 一个很棒的虚拟专用网是你新的最好的朋友

有很多虚拟专用网络提供商在那里吮吸并窃取你的数据出售给第三方，但如果你能 [找到一个可信的](https://lifehacker.com/how-to-choose-a-vpn-1831320407)——或者如果你创建自己的——这是一个很好的方式来提高你访问公共 wifi 时的安全性。确保你在做任何事情之前打开*，比如说，提交一个网站或服务的登录名和密码。并不时对其进行 [DNS 泄露](https://pixelprivacy.com/resources/dns-leak/) 测试。*

#### 确保你也堵住了任何其他的安全漏洞

无论您是使用公共 wifi 还是在家浏览，在您的浏览器和操作系统中安装各种有用的工具，帮助您避免网络问题。拥有强大的 [杀毒和](https://lifehacker.com/why-you-should-use-windows-defenders-ransomware-prevent-1837311176)反恶意软件设置。在你的浏览器中使用 [UBlock Origin，Privacy Badger，和 HTTPS Everywhere](https://lifehacker.com/am-i-screwed-if-i-dont-upgrade-windows-7-by-january-15-1840903120) 。确保您没有共享计算机上的文件夹，包括常用的 Windows 公共文件夹。使用最新的安全补丁更新您的系统和应用。注意浏览器地址栏中显示的内容。当你使用完公共 wifi 后，忘记网络，这样别人就不能欺骗它，让你的电脑或手机再次登录(在你启用 VPN 之前)。

HTTPS 很棒，也很重要——不要误会我的意思——但如果认为这是你在连接公共 wifi 时唯一需要考虑的事情，那也同样是错误的。它有助于让你更安全；它不会让你安全。
