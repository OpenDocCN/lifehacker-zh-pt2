# 你最好的黑客体验是什么？

> 原文：<https://lifehacker.com/what-was-your-best-hackintosh-experience-1834388817>

Lifehacker 最受欢迎的帖子之一，我们的 [黑客指南](https://lifehacker.com/the-always-up-to-date-guide-to-building-a-hackintosh-o-5841604?_ga=2.245553159.2031387578.1556546056-1723114163.1524514905) ，开始显示出它的年龄。是时候给这篇文章涂上一层新的油漆了，但在改造工作开始之前，我想听听你对每个极客的大梦想的体验:在非苹果设备上运行 macOS 的能力。



出于两个原因，我请求你们的故事、评论、建议和提示。首先，我已经有一段时间没有构建一个 hackintosh 了——一个*长的*而——我记得它需要 [一点配置](https://io9.gizmodo.com/how-to-run-mac-os-x-inside-windows-using-virtualbox-5938332) 才能开始。我敢肯定，有很多资源可以提供最新的信息和应用程序，帮助人们开始使用 hackintosh。**我很想听听你是从哪里获得信息、应用程序和实用程序来让这个神秘的过程发挥作用的。**

(毕竟， [并不是大多数人想要构建的](https://news.ycombinator.com/item?id=17708375)*的东西，尽管我们许多人都梦想在非 Mac 电脑上运行信息。总有一天，苹果。总有一天。)*

*第二，我记得整个过程花了很多时间进行研究和配置。即使经历了这一切，我记得我对结果并不感到兴奋。不可否认，我选择了“懒惰”的路线，转而安装了一台运行 macOS 的 Windows 虚拟个人电脑——不完全是一台 hackintosh，但如果遇到挫折，可以很容易地使用和删除。*

*尽管如此，我的台式电脑性能相当高，这就是为什么我有点惊讶地发现，在虚拟环境中运行 macOS 的体验是滞后和不愉快的。以至于我记得我想我宁愿穿过房间，等待我的旧 MacBook 加载，而不是处理虚拟版本。*

*也许是我？也许虚拟 PC 的路线终究不是一条路要走？也许自从我最后一次尝试以来，构建一个 hackintosh 已经有所改进了？不管怎样，我很想听听你的经历——什么对你有效？你的建筑哪里出错了？你运行的是什么系统规格？ [还有人再黑](https://www.youtube.com/watch?v=JbLKbcVA7Ko) 吗？*

*请在评论中告诉我，我将开始计划我的非 Mac 版本— [，至少在我还能](https://gizmodo.com/apple-reportedly-killing-laptops-with-intel-inside-the-1824262384?_ga=2.182499109.2031387578.1556546056-1723114163.1524514905) 的时候。*
