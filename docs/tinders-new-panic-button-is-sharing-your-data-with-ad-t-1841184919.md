# Tinder 的新紧急按钮是与广告技术公司分享你的数据

> 原文：<https://gizmodo.com/tinders-new-panic-button-is-sharing-your-data-with-ad-t-1841184919>

Tinder 为一些被指控强奸的--[比](https://www.tampabay.com/news/crime/2019/10/24/man-shot-in-davis-islands-case-now-accused-of-raping-tinder-date/) - [恒星](https://www.washingtonpost.com/nation/2019/09/19/an-alleged-serial-rapist-met-victims-bumble-tinder-his-attorneys-say-women-were-there-hook-up/) [男性](https://www.stuff.co.nz/national/crime/117973796/man-denies-raping-tinder-date-and-offering-her-money-to-keep-quiet) 提供了一个约会平台——在一个可怕的案例中， [肢解](https://www.cnn.com/2019/06/25/us/defendant-murder-slash-throat-court/index.html)——他们通过该平台认识的女性。但即使公司做了正确的事情，仍有隐私权衡要考虑。



虽然该公司似乎仍然缺乏一些基本的安全措施，比如说， [先发制人地筛查已知的性犯罪者](https://www.buzzfeednews.com/article/hillaryflynn/tinder-lets-known-sex-offenders-use-the-app-its-not-the) ，但该公司*周四宣布了最新的努力，以遏制其多年来积累的声誉: [“应急按钮”](https://gizmodo.com/tinder-is-getting-a-panic-button-specifically-tailored-1841179910) ，将每个用户与紧急响应者联系起来。在一家名为 Noonlight 的公司的帮助下，Tinder 用户将能够分享他们的约会细节——以及他们的特定位置——以备执法部门需要介入。*

*一方面，该声明是积极的一步，因为该公司试图解决其用户群中最糟糕的角落。另一方面，正如 Tinder 在给 Gizmodo 的一封电子邮件中确认的那样，Tinder 用户*将*需要下载单独的免费 Noonlight 应用程序，才能在 Tinder 的应用程序中启用这些安全功能——正如我们已经看到的 [时间](https://gizmodo.com/dont-just-uninstall-old-apps-delete-your-accounts-as-we-1836237449) 和 [时间](https://gizmodo.com/dating-apps-caught-sharing-your-details-with-dozens-of-1840997236) (以及 [时间](https://gizmodo.com/these-apps-reportedly-shared-sensitive-personal-informa-1832827887) 和 [时间](https://gizmodo.com/researchers-create-fake-profiles-on-24-health-apps-and-1833474535) )一样，免费应用程序在设计上并不是很好*

*不出所料，Noonlight 的应用也不例外。通过下载该应用程序并监控发送回其服务器的网络流量，Gizmodo 发现了广告技术领域的一些主要名字——包括脸书和谷歌旗下的 YouTube——每分钟都在收集关于该应用程序的细节。*

*“你知道，对这种东西持怀疑态度是我的工作——但我还是有点被愚弄了，”电子前沿基金会的技术专家贝内特·塞弗斯说，他专注于广告技术的隐私含义。“他们把自己推销成一种‘安全’工具——‘智能现在是安全的’是他们网站上问候你的第一句话，”他继续说道。“整个网站旨在让你感觉有人在照顾你，你可以信任。”*

*在 Noonlight 的辩护中，实际上有一大批值得信赖的第三方，可以理解的是，他们应该从应用程序中收集数据。 [正如该公司的隐私政策](https://www.noonlight.com/privacy) 所规定的，当执法部门的某人试图将你从危险境地中解救出来时，你的准确位置、姓名、电话号码，甚至与健康相关的情报都将派上用场。*

*不太清楚的是，他们保留与“匿名”第三方合作的权利。该政策指出:*

> *当您使用我们的服务时，您授权我们与相关紧急响应者共享信息。此外，我们可能会共享信息[...]与代表我们提供服务或帮助我们提供服务(如会计、管理、技术、营销或分析服务)的第三方业务合作伙伴、供应商和顾问合作。"*

*当 Gizmodo 联系 Noonlight 询问这些“第三方商业合作伙伴”时，一位发言人提到了该公司与主要品牌之间的一些合作伙伴关系，如其与 Fossil 智能手表 的 [2018 整合。当被问及公司的营销合作伙伴时，这位发言人——据这位发言人说，还有公司的联合创始人——最初否认公司与任何合作伙伴有过合作。](https://www.digitaltrends.com/mobile/noonlight-and-fossil-add-emergency-mode-to-smartwatches/)*

*根据 Gizmodo 自己对 Noonlight 的分析，我们发现至少有五个合作伙伴从这个应用程序中收集了一些信息，其中包括脸书和 YouTube。另外两个，分别是 [分支](https://branch.io/) 和 Appboy(后来改名为 [钎焊](https://www.braze.com/) )，专门连接给定用户在他们所有设备上的行为，以达到重定向的目的。 [Kochava](https://www.kochava.com/) 是 [各种受众数据](https://www.kochava.com/data-marketplace/#audience-targeting) 的主要集散地，这些数据来自不计其数的应用程序。*

*在 Gizmodo 透露我们已经分析了该应用的网络，并且网络数据显示有第三方存在之后，Noonlight 的联合创始人 Nick Droege 通过电子邮件提供了以下内容，大约在该公司强烈否认存在任何合作关系的四个小时之后:* 

> *Noonlight 使用 Branch 和 Kochava 等第三方只是为了了解标准用户归属和改善内部应用内消息传递。第三方收到的信息不包括任何个人身份数据。我们不会出于营销或广告目的向任何第三方出售用户数据。Noonlight 的使命一直是保证我们数百万用户的安全。*

*让我们解决一下这个问题，好吗？应用程序是否真的向这些第三方“出售”用户数据是一个完全棘手的辩论，甚至在加州消费者隐私法案——或 CCPA——[于今年 1 月生效](https://gizmodo.com/the-chaos-of-californias-new-privacy-law-has-already-be-1840730477) 之前，这个辩论就已经在董事会、新闻编辑室和法庭上展开了。*

*在这种特殊情况下，*清楚的是，即使数据没有被“出售”，它*也在*与相关的第三方易手。例如，Branch 收到了手机操作系统和显示屏的一些基本规格，以及用户下载应用程序的事实。该公司还为这款手机提供了独特的“指纹”，可以使用 [将用户与他们的每台设备](https://branch.io/glossary/device-fingerprint/) 联系起来。**

*与此同时，脸书通过其 Graph API 收到了类似的关于设备规格和下载状态的基本数据 [，谷歌通过其](https://developers.facebook.com/tools/explorer/v2/) [Youtube 数据 API](https://developers.google.com/youtube/v3/docs) 收到了这些数据。但即使这样，因为我们在谈论，嗯， [*脸书和谷歌*](https://www.emarketer.com/content/facebook-google-duopoly-won-t-crack-this-year) ，很难说最终会从甚至*那些*基本数据点中榨出什么。*

*应该指出的是，即使没有 Noonlight 集成，Tinder 在历史上也曾与脸书共享数据，另外 [收集关于你的数据](https://www.theguardian.com/technology/2017/sep/26/tinder-personal-data-dating-app-messages-hacked-sold) 。*

*至于这位联合创始人声称被传输的信息不是“个人可识别”的信息——比如全名、社会保险号、银行账号等等。，统称为 PII——考虑到我们观察到的流传的规范实际上是多么基本，这似乎在技术上是准确的。但是个人信息并不像有些人想的那样被大量用于广告定位。不管怎样，非 PII 的数据可以被交叉引用，以建立特定的个人档案，尤其是当涉及到像脸书这样的公司时。*

*至少，这些公司中的每一家都在窃取关于该应用的安装和安装该应用的手机的数据——对于那些习惯了从他们的到 [的病史到他们的性取向](https://gizmodo.com/dating-apps-caught-sharing-your-details-with-dozens-of-1840997236) 被交给营销人员以获取利润的读者来说，这可能看起来相对温和，特别是考虑到 noollight*也*要求随时打开位置跟踪。*

*但正如塞弗斯指出的那样，这最终与主题无关。* 

*他解释说，“把它看做‘你分享的伙伴越多，情况就越糟’是不正确的。”“一旦它脱离应用程序，进入一个想要从中赚钱的营销人员手中，它可能在任何地方，也可能无处不在。”*

*当看到像 Kochava 这样的合作伙伴时，这是一个值得思考的问题。Koch ava 在收集关于你的手机操作系统的类似基本信息的同时，也是一家乐于吹嘘其“ [数百个广告网络和出版商合作伙伴](https://github.com/Kochava/kochava-tracker-ios-sdk-cocoapod) ”的公司因为广告的指挥链是 [不仅仅是一点点不透明](https://www.adweek.com/programmatic/how-advertisers-are-untangling-the-programmatic-supply-chain/) ，这几百人中的某个百分比完全有可能在一个针对非常特定(并且非常脆弱)的人群的应用上获得这些数据——即使他们不应该这样做。*

*换句话说，*有人下载这个应用程序的纯粹事实*至少是一个暗示，他们可能是一个女人，可能害怕成为 [另一个统计](https://www.rainn.org/statistics/victims-sexual-violence) 。沿着这条线的某个地方，这些基本数据可以用来锁定那些下载这个带有 [某种自卫钥匙链](https://yr.media/news/amazon-self-defense-products-for-women-because-prime-shipping-is-faster-than-changing-rape-culture/) 广告的特定应用程序的人。 [或咨询服务](https://www.natlawreview.com/article/direct-to-consumer-dtc-telehealth-does-industry-need-more-regulation-podcast) 。 [或](https://fidelitasdevelopment.com/insight/5-ways-firearm-retailers-can-advertise-online-without-facebook-ads/) 一枪。因为谁知道呢，他们可能需要这些东西，对吧？*

*正如塞弗斯所说，“那些被迫下载的人正是那些被他们分享的数据置于最大风险的人”，这是绝对正确的——这适用于他们整个数字生活的数据，包括他们下载的应用程序。*

*插入 Noonlight 的每一个人——以及每一次创伤、每一次恐惧、每一次痛苦的遭遇——最终都可能被归入一个“下载了这个特定应用的人”的桶中，而这个桶将成为数字广告生态系统中其余可定位数据点中的一个亮点。然而最终，不是什么进入了这个特殊的信号，或者这个信号的大小，是站不住脚的——而是这个信号确实存在。*
