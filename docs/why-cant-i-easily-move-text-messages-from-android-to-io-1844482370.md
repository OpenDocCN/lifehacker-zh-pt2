# 为什么我不能轻松地将短信从 Android 移动到 iOS？

> 原文：<https://lifehacker.com/why-cant-i-easily-move-text-messages-from-android-to-io-1844482370>

我不记得上一次我在 Android 和 iOS 之间切换我的日常驱动程序是什么时候了——我用手机处理我所有的个人物品，而不仅仅是我用手机处理各种与 Lifehacker 相关的事情。虽然技术已经取得了长足的进步，但我的假设是，将一个手机的*全部*内容转移到另一个基于竞争平台的仍然是一场噩梦。

Watch

不幸的是，life hacker reader**Liseetsa**正在亲身体验这种快乐，我担心我能给她的建议很少能真正解决她的问题:如何将她的 Android 信息迁移到 iOS。正如她告诉 [理工 911](https://lifehacker.com/c/tech-911) :

> “我们中的一些人几乎像使用数据库一样使用信息。我有一个同事团队，他们多年来每天都用照片(彩信)发送截屏和信息。我通常不会将它们转发到电子邮件帐户，也不会将文本/照片保存到手机文件/备份卡中。苹果销售人员承诺，他们向 iOS 应用程序的转移可以处理从 Android 4.0 或更高版本到 iPhone 的彩信传输。
> 
> 几周后，他们仍在处理应用程序中的“错误”,这些错误从邮件中剥离我的附件，并将其转储到一个自发创建的线程(从我到我)中，该线程只包含图像，没有任何关于发送者或发送时间的指示。就像你奶奶衣柜里的一鞋盒照片。他们坚持要我去 COVID 世界找一家授权经销商，将 Android 操作系统从 6.0 升级到 8.0——他们说这就是问题所在，在我的系统升级之前他们无能为力。然后他们运行日志，发现了“应用程序中的一个错误。”他们说需要几天时间来修复，然后会给我回复。
> 
> 上周，我开始收到他们试图联系我的消息——希望是因为他们修复了漏洞。但是链接从来不工作，我已经和“低层技术人员”等了几个小时，等待高层技术人员告诉我工程师们是否已经修好了。我应该再次尝试转接吗？"

### 移动邮件比你想象的要复杂得多

如前所述，我还没有做这个举动...太久了。由于我使用 iPhone 进行日常互动，向你介绍如何将信息上传到安卓系统不会有太大帮助。(毕竟你走的是相反的路。)

我最好的建议听起来有点笼统，所以在我们进入一些你可能不太喜欢的解决方案之前，让我先从这个开始。在你做任何与转账相关的事情之前，确保你在你的 [iPhone](https://lifehacker.com/how-to-update-your-iphone-and-all-your-apps-1843480748) 和你的 [Android](https://lifehacker.com/how-to-update-your-android-device-and-apps-1843395131) 上使用最新的操作系统和应用程序。不能保证你能以任何一种方式传输你的信息，但是使用最新版本的操作系统会给你最好的机会将你的信息从一个平台迁移到另一个平台。

除此之外，我假设你已经采纳了苹果的建议，并查看了其简单易懂的网站 ，以便从 Android 迁移到 iOS。你要特别注意“ [**”移动到 iOS**](https://play.google.com/store/apps/details?id=com.apple.movetoios) ”应用程序，这个*可以*帮助你，但并不总是像它的名字暗示的那样有用。我读过很多人的报告，他们说这个看似简单的应用程序就是不工作，或者即使能工作也要花很长时间。

如果你在那里没有找到任何运气，你的运营商可能有一个你可以尝试的应用程序。每个 t 何大三—[**AT&T**](https://www.att.com/features/mobile-transfer/)[](https://www.verizon.com/support/apple-android-transfer/)****和[T-Mobile](https://www.t-mobile.com/resources/how-to-do-a-content-transfer)—都应该提供自己的免费转账 app。还是那句话，不能保证它们中的任何一个都完全适合你的情况，但是我见过人们使用其中一个比使用 iOS 更成功。****

### ****也许放弃是件好事？****

****我认为你最好的选择是弃船。这并不意味着扔掉你 Android 上的大量文本历史记录。相反，使用一个类似于 [**短信备份&恢复**](https://play.google.com/store/apps/details?id=com.riteshsahu.SMSBackupRestore&hl=en_US) 的应用程序，用它将你的信息导出到 [一个 XML 文件](https://github.com/KermMartian/smsxml2html) ，然后你可以将它转换成一个更容易阅读的 HTML 文件。(还有其他的[app](https://play.google.com/store/apps/details?id=com.gilapps.smsshare2&hl=en_US)或许可以一次搞定这一切，但是我还没有测试过。)****

****然后，您可以将该文件和任何附件上传到云存储服务。如果你不得不回顾你在以前的文章中说过的话，请随意在你的新 iPhone 上浏览这个文件。****

****这是一个理想的解决方案吗？不，一点也不。在一个完美的世界里，将你的信息转移到一个新的平台是非常容易的。不过，你真的想这样吗？假设有人得到了你的设备或者这个虚构的、跨平台的云信息服务的登录信息。你真的*想让*他们有一个快速的方法将你的整个信息历史完全导出到另一个平台吗？大概不会。如果你的信息都是通过云连接的，并且容易在其他平台之间来回导出/导入，我怀疑黑客攻击的可能性会更令人不快。****

****这种合作设置也需要谷歌和苹果好好合作，我怀疑也没有兴趣提供一种简单的方式将你的信息从他们的平台转移到竞争对手的平台。****

****所以，不幸的是，你有点卡住了。如果这些更简单的解决方案对你不起作用，我也不会让你的安卓信息传到 iOS——如果这有可能的话。老实说，我认为“必要时备份和审查”技术是正确的方法。我曾经非常担心在不同的安卓手机之间切换时如何保存我手机的信息，直到我最终意识到我很少————搜索旧短信。****

****把短信当作短暂的通知，把你必须保存的对话放在另一个可以普遍访问的平台上*可以让你保存相当长的历史。这可能不像 Slack(除非你有一个有很好的保留政策的公司账户)，但可能像共享文件夹/笔记/相册一样简单。或者，除此之外，每个人都可以访问和更新的云文档。*****

*****虽然你可能无法将旧信息保存到新 iPhone 上，这是一件令人失望的事情，但这一举措给你提供了一个机会，让你重新思考你和你的同事是如何沟通的。如果这能帮助你找到一个更好的、经得起未来考验的、与平台无关的解决方案，那么丢失一些消息可能是值得的。*****
