# 发电子邮件“只是打个招呼”真的有那么糟糕吗？

> 原文：<https://lifehacker.com/is-emailing-just-checking-in-really-that-bad-1829841607>

在工作和生活中，我喜欢剧本。如果重复类型的互动是你生活的一部分——进行某种闲聊，必须以一定的频率发送相同类型的电子邮件，每天给客户、供应商或消息来源打电话——当你知道有一句妙语可以让一个啰嗦的人挂断电话时，为什么每次都要浪费精力想出新的材料呢？或者在聚会上有效地向人们解释你到底是做什么的？【T2

Watch

我曾经为一个人工作过，他给每个场合的每个客户都送了完全一样的礼物，他的电子邮件是如此的统一和简洁，就像是机器人写的一样。起初我觉得这很奇怪，但后来我开始欣赏它的本来面目:这是一种高超的效率，更不用说是一种在传球时阻止令人困惑或潜在不舒服的互动的策略。

显然这里有限制——有些短语是众所周知的祸害(“见到你真高兴！”“确定你看到了吗？”)并需要立即从我们的集体文化词汇中删除。

 [https://lifehacker.com/embed/inset/iframe?id=twitter-943876799221596160&autosize=1](https://lifehacker.com/embed/inset/iframe?id=twitter-943876799221596160&autosize=1) 

但是，我不*不*认为“只是签到”是其中之一。我感觉 [受到了最近这篇关于金钱的文章的人身攻击，这篇文章建议不惜一切代价做各种修辞性的后空翻，以避免“只是签到”](http://time.com/money/5427668/the-three-words-you-should-never-use-in-a-work-email-and-what-to-say-instead/) 。这篇文章在 Lifehacker 的员工中引发了一场激烈的争论，争论的焦点是“直接登录”是被动的还是有效的。

更重要的是，如果它如此可怕，你会用什么来代替它？关于你为什么又发邮件的杂乱解释？“想检查一下这个的状态”？(众所周知，我也用过这个，但感觉明显更具攻击性。)*“绕回来”？？？？*
