从哪里开始。我想我会从最明显的建议开始:如果你担心公司跟踪你的行踪，我有坏消息要告诉你。你的设备、你的应用和你的服务都被明确地设计来找出你在哪里，你在做什么，以及如何利用这些来为你提供更相关的广告和内容。我们已经广泛地讨论过了，众所周知，公司有各种各样的工具来大致了解你是谁，你在哪里，你喜欢在你的设备上做什么——如果不是互联网的话。这是你享受所有免费服务的代价。

所以，如果你非常关心隐私，你的第一反应不应该是冠状病毒。你或许应该换一部更笨的手机，或者至少尽你所能锁定你手机的设置，这样你就可以尽量少和这些公司分享你的信息。这不会阻止他们继续收集你的信息，但至少会减少他们能做的事情，如果没有别的，会让你感觉好一点。

### 暴露通知 API 的实际工作方式

现在，回到新冠肺炎。不，你的智能手机没有 COVID 传感器或内置的 COVID 跟踪器。通过更新操作系统，它可以拥有的是 [一个新的暴露通知 API](https://developers.google.com/android/exposure-notifications/exposure-notifications-api)。如果你的手机里有这个，而且只有这个，从现在到前疫情时代，你的隐私没有什么真正的改变。

让我们再深入一点。暴露通知 API 要求您的设备至少运行 iOS 版本 13.5 或 Android Marshmallow(版本 6)。它本身是无辜的。谷歌和苹果并没有用它来侦查你是否已经被冠状病毒感染——通过你的短信，你的健康信息，你的电子邮件，或者任何其他的阴谋论。政府也不会窃听你的电话来了解你的健康状况或追踪你的精确行动。(我希望。)

要使用上述暴露通知 API，您需要下载一个应用程序来利用它。否则，API 默认不做任何事情。目前，只有少数这些应用程序在美国可用，而且它们都与特定的地理位置相关联。换句话说，比如说，下载并安装弗吉尼亚州的 [应用程序](https://apps.apple.com/us/app/id1518059690) 使用 API，对你在加州的生活没有太大好处。

但即使你*确实*为你的位置下载了一个应用程序，你仍然必须选择加入以授予所述应用程序访问 API 的权限。如果你这样做了，那么当附近有其他人使用该 API 时，你的手机将能够通知你。那声音有点像*“OMG 可怕的位置追踪 ahhghuhgh1！1!，"*我意识到了，所以我们来解开这是什么意思。正如谷歌中[描述的](https://www.google.com/covid19/exposurenotifications/) :

> *   *"This technology only works if you decide to opt in. If you change your mind, you can always turn it off.*
> *   *The exposure notification system will not collect or use location information from your device. Using Bluetooth, it can be used to detect whether two devices are close to each other without revealing the location of the devices.*
> *   *All exposure notification matches happen on your device. The system will not share your identity with other users, Apple or Google. Public health authorities may ask you to provide additional information, such as telephone number, in order to contact you for additional guidance.*
> *   *Only applications in public health departments can use this technology. Their applications must meet specific standards in terms of privacy, security and data usage. ”*

当然，在社交媒体上发布垃圾科学的人可能没有兴趣听负责曝光通知 API 的公司之一谈论它是如何工作的——阴谋论就是这样。但事实上，这就是它的工作方式。

安装一个使用 API 的应用程序不会突然将你的位置发送到一个巨大的数据库，然后用来跟踪你的行踪；全副武装的医生不会踢开你的门，把羟氯喹药片塞进你的喉咙。你的健康保险不会突然提高你的费用，因为不知何故，神秘地，他们发现你已经或已经暴露于冠状病毒。

### 如果有人得了新冠肺炎，包括你，会怎么样

如果你接触到一个患有新冠肺炎的人，以下是如何通知你以及哪些数据 [可能会被共享](https://support.google.com/android/answer/9888358) :

> *“一旦您选择加入暴露通知系统，它将在您的设备上生成随机 id。为了帮助防止跟踪，你的手机随机 ID 每 10-20 分钟改变一次。*
> 
> *您的手机在后台工作，通过蓝牙与您周围也开启了暴露通知的手机共享这些随机 id。当您的手机检测到来自其他设备的随机 ID 时，它会将该 ID 记录并存储在您的设备上。*
> 
> 如果有人报告患有新冠肺炎，并且他们的 ID 存储在您的手机中，您的应用程序会通知您接下来要采取的措施。"

> “政府公共卫生当局决定哪些因素可能表明暴露。
> 
> *如果你的应用程序得知你与自称患有新冠肺炎的人联系过，系统可能会与应用程序共享信息，包括:*
> 
> *   联系发生的那天。
> *   *接触持续了多久。*
> *   *该联系人的蓝牙信号强度。*
> 
> *您的公共卫生机构应用程序不允许使用您手机的位置。*
> 
> 暴露通知系统本身不会使用你的位置，也不会与应用程序、谷歌或苹果共享其他用户的身份。"

如果*您*被感染，您将负责在您使用的任何应用程序中自我报告此状态。你可以是个混蛋，什么都不说；你可以直接卸载这个应用。你想做什么都可以。只有你个人的是非感受到威胁——你的医疗服务提供者不会出卖你，把你的所有信息，包括你的实时位置，发送给使用你所在州提供的任何合同追踪应用程序的其他人。

(我知道这听起来很傻，但我只是走在了“监视 meeeee”人群的前面。)

自我报告你的状态不仅仅是“点击一个按钮，现在我有 COVID”这种交易——至少，对于那些对公众有用的应用程序来说不是。正如弗吉尼亚的 COVIDWISE 所描述的:

> 所有新冠肺炎病毒检测呈阳性的人的实验室结果都被送到[弗吉尼亚州卫生部]。这与应用程序无关。我们的工作人员根据实验室报告中提供的信息对报告为阳性的人进行跟踪。出于对所有应用程序用户的礼貌，VDH 将验证阳性测试，然后向 COVIDWISE 用户提供个人识别号(PIN)。您必须使用该 PIN 才能向应用程序报告阳性结果。这可以防止人们误报阳性结果，从而产生错误的暴露通知。VDH 希望所有的应用程序用户都有信心，当通过应用程序接收到可能的新冠肺炎暴露时，这是一个真实的事件。”

### 无论如何，大多数人不会在手机上使用合同追踪

所以，就这样了。虽然智能手机隐私有很多问题，但我不认为暴露通知 API 是你想死的地方。我也不认为这将是*的*大交易，因为根本没有足够的普遍授权每个人都使用他们的手机来帮助对抗新冠肺炎暴露。

但即使这样，我们也会陷入一个尴尬的隐私交易。虽然我绝对希望谷歌和苹果在所有手机上强制使用这种 API，甚至在我们处于这种恼人的疫情中时，在每个人的设备上激活相应的(以隐私为重点的)应用程序，但*这*将是一个问题。即使这一举措是为了公共利益，但仍会让人觉得是不自量力。我可以理解一种集体的担忧，甚至是基于对技术工作原理的完全误解，围绕着授权的数字跟踪来打击极端愚蠢的冠状病毒。

当然，我们让各种应用程序定期从我们的设备中吸取数据，这似乎并没有真正困扰大多数人。也许我想得太谨慎了，一个联系追踪 API 和应用程序不会比一个全新的社交网络或流行游戏更令人担忧。如果这些没有给你隐私方面的恐惧，你为什么会如此担心那些试图保持你健康和安全的东西呢？
