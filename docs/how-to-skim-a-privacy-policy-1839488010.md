# how to skim a privacy policy 1839488010

> 原文：<https://lifehacker.com/how-to-skim-a-privacy-policy-1839488010>

你刚刚下载了最热门的新应用程序，现在你必须在使用它之前点击通过样本文本页面。谁有时间做这些？即使是注重隐私和安全的人也经常浏览应用程序的服务条款，而且理由充分。卡耐基梅隆大学 [的研究人员估计](https://kb.osu.edu/bitstream/handle/1811/72839/ISJLP_V4N3_543.pdf?sequence=1&isAllowed=y) 要通读一年中遇到的所有隐私政策需要花费多达 [76 个工作日](https://www.theatlantic.com/technology/archive/2012/03/reading-the-privacy-policies-you-encounter-in-a-year-would-take-76-work-days/253851/) ，那还是在 2008 年！

虽然你不应该感到被迫逐字阅读你的应用程序和服务的隐私政策——无聊！当你略读的时候，仍然有一些关键的标准你应该寻找。对，略读；你不应该完全忽视隐私政策，因为知道你的数据正在被做什么是很重要的。

### **较短的隐私政策并不总是更好**

多亏了[【GDPR】](https://lifehacker.com/whats-the-deal-with-gdpr-and-your-online-privacy-1826314658)和[【CCPA】](https://www.caprivacy.org/)，许多公司开始用更容易理解的语言写出他们的隐私政策。有些人甚至关注数据如何收集、存储、使用和共享等问题，但这并不总是有助于简洁。

Brex 的隐私和数据顾问 Whitney Merrill 说，长期的隐私政策不一定是坏的。“如果人们看到非常长的东西，他们会想，‘哦，我的上帝，你在做所有这些坏事！’但这也可能意味着你对一切都超级透明，你试图解释正在发生的事情，”她说。

换句话说，不要给一个隐私政策很短的公司免费通行证，也不要忽视一个有点长的隐私政策——在这种情况下，彻底性并不是世界上最糟糕的事情。

### **寻找简洁、更新和回调**

如果简洁不一定是更好的隐私政策的标志，那么什么是呢？使用易读语言的隐私政策，或者简明扼要的隐私政策，至少表明该公司正试图以用户能够理解的方式进行沟通。隐私政策何时更新的信息也是一个好迹象。(GitHub 甚至在其 [网站策略库](https://github.com/github/site-policy/) 中描述了其隐私策略的变化。)

美林喜欢苹果的 [隐私页面](https://www.apple.com/privacy/) ，说明了它的隐私价值观和原则。她指出，苹果有一个两个小蓝人握手的图形，这个图形出现在他们的注册流程和副本中。每当苹果的应用程序要求以某种方式使用你的个人信息时，这个图标就会出现在 iOS 和 macOS 中。

“任何时候你看到他们，都表明他们将谈论数据、数据使用、数据共享或数据收集，”她说。

在某些地区，苹果甚至会提供链接或参考他们的政策，以提供关于正在收集的内容和使用方式的更多信息。“我认为这也很有帮助，因为其中一个问题是，当你给某个东西授予权限时，你并不真正了解它的用途，”她说。

### **你可能根本不需要深究隐私政策**

GitHub 的公司法律顾问 Fred Jennings(他要求澄清他在与我们交谈时并没有提供法律建议或担任 GitHub 的律师)指出，你对隐私政策的审查程度可能取决于你使用该应用程序或工具的目的。“在你开始阅读政策之前，最关键的问题是，我向这项服务提供了什么样的数据，它有多隐私？”他说。

例如，一个担心版权的摄影师可能会比一个只是在 Instagram 上发布业余美食照片的人更关注照片工具的政策，一个企业主可能会密切关注云共享或文件存储服务。

### **宽泛可不好**

一些公司起草非常宽松的条款(对他们自己)，通常是因为他们的律师想涵盖所有可能的情况。在某些情况下，公司实际上并没有使用你提供的数据，但他们希望将来有机会这样做，而不会受到公司在改变隐私政策时有时会受到的负面影响(或者需要再次获得所有人同意的麻烦)。然而，很难判断公司是否打算，例如，利用你签字放弃的你自己的用户创作内容的版权，或者把它作为语言包括在他们的政策中以防万一。

斯坦福大学法学院互联网与社会中心消费者隐私部主任 Jen King 说，隐私政策中使用的语言经常故意含糊不清。例如，许多形式说他们“可能”或“可能”或“可能”做某些事情，这使得解释变得棘手。

当有疑问时，最好假设公司正在做或即将做的正是他们在政策中保留的权利。对你来说，这是否是一个交易破坏者取决于你使用他们的服务做什么。根据 Jennings 的说法，在浏览政策时，你可能需要注意的一些领域包括公司收集哪些数据，以及如何共享和存储这些数据。

### **数据收集**

一项伟大的隐私政策阐明了公司正在收集你的哪些信息。这可能包括您的姓名、地址、社交媒体详细信息、IP 地址和其他唯一标识符、GPS 信息等等。这些数据也可能与公司从第三方收集的信息相结合。

<label class="bxm4mm-13 juykRM">Advertisement</label>

除了你的个人信息，还有你使用公司的应用或服务创建的内容的问题。注意该公司是否获得了将你的图片或其他内容发布给第三方的许可，或者它是否要求更大程度的所有权。如果摄影师计划将图像发布到照片共享网站，但稍后可能还想在其他地方出售，则发布者可能想知道是否有任何其他实体有权发布该作品。詹宁斯解释说:“如果我不得不删除我的账户，说我没有在其他任何地方发布过它，我仍然想知道这些数据是否永久许可给该公司，即使在我删除我的账户之后。”

首先点击 Ctrl+F 并寻找“所有权”,找出公司对你的创造性作品拥有何种权利和何种许可:所有权、永久所有权(永久或永久是法律关键词，有时用来解释某样东西持有多长时间)、全球所有权或单独决定权。一旦你找到合适的部分，花些时间仔细阅读。

### **数据共享**

“当我浏览他们的时候，我寻找的第一件事是我是否能确定他们是否在出售我的数据，”金说。这也是声明性语句适合的地方。“偶尔你会发现一项隐私政策，该公司在这个问题上已经表明了立场，他们只会说，很明显，我们不会出售或与任何人分享你的数据。”

<label class="bxm4mm-13 juykRM">Advertisement</label>

但是，即使那些坚持不出售你的数据的公司也做得不够，因为“分享”也同样糟糕。那些写着“我们只与我们的会员分享”的应用程序可能会比你想象的更广泛。例如，Match Group，Inc .拥有 Match，也拥有 Tinder、OkCupid 和 plenty offish——OkCupid 用户可能不知道他们的数据正在与 Tinder 共享。此外，即使您信任您想要与之共享信息的组织的数据共享政策，与之共享信息的组织*也可能有不同的政策*

寻找“收集”、“分享”和“分支机构”这些词。

数据共享还包括与政府共享信息。詹宁斯说:“如果这是我存储更多私人数据的地方，我可能也会对那里的‘法律程序’或‘执法’等短语感到好奇。”“公开”也可能是有用。

### **数据存储**

使用 Control-F 键查找“存储”和“加密”等短语，以查找有关公司如何将您的数据保存在服务器上的信息，并查找“删除”或“保留”以查看您可以了解到它将您的信息保留多长时间。

<label class="bxm4mm-13 juykRM">Advertisement</label>

不幸的是，关于数据存储的信息往往模糊不清，公司遭受了如此多的安全漏洞，以至于很难统计。即使您对隐私政策感到满意，保护您的数据也需要额外的措施，而不仅仅是信任存储数据的公司:

*   考虑只为应用程序提供运行所需的信息。添加超出需求的数据并不总是能带来很多好处。例如，一个周期跟踪应用程序只需要日期和时间就可以工作；它不需要你性伴侣的名字和生日。
*   请确保在您的帐户上使用唯一、复杂的密码，这样，如果您的密码被泄露，它只会影响一个帐户。
*   除了您的密码之外，使用安全密钥(如 Yubikey)或类似于[Google Authenticator](https://support.google.com/accounts/answer/1066447?co=GENIE.Platform%3DAndroid&hl=en)或 [Authy](https://authy.com/) 的应用程序进行多因素认证。
