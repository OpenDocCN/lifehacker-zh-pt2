# 使用文本到语音阅读器检查书写错误

> 原文:[https://life hacker . com/use-a-text-to-voice-reader-to-check-for-written-mission-1833668053](https://lifehacker.com/use-a-text-to-voice-reader-to-check-for-written-mistake-1833668053)

当你想在求职信或书面材料中给人留下好印象时，你会非常沮丧地发现你在第一句话中漏掉了一个单词。我们的大脑并不总是能捕捉到简单的错误，所以争取一点数字帮助也无妨。

[u/turtles4dayz](https://www.reddit.com/user/turtles4dayz) 在 r/LifeProTips 上建议，任何担心拼写或语法错误的人都应该 [将他们的文本通过文本到语音阅读器](https://www.reddit.com/r/LifeProTips/comments/b6qxcr/lpt_before_you_send_out_a_cover_letter_use_a/) :

> 在你发出求职信之前，使用文本语音阅读器来检查打字错误。
> 
> 没有什么比发出求职信后发现少了一个单词或一点标点符号更让人难过的了，尤其是如果你——我们都必须这样——声称自己“注重细节”你可能会错过这些，因为你的大脑会把它看到的纠正为你想说的，但机器人不会。

一位评论者问拼写检查是否基本上做同样的事情，u/turtles4dayz 有一个很好的答案:

> 拼写检查不能捕捉丢失的单词。例如:“作为一名研究生，我已经就这些主题写了很多论文。”有一个遗漏的介词不会出现，因为你知道你想说的是“关于这些主题的论文”，你可能看不到它。这种事经常发生。

的确如此，而且这是个好主意！网上有很多这样的应用，其中一个我用来测试 u/turtles4dayz 的建议；这就叫 [自然读者](https://www.naturalreaders.com/online/) 。你只需将一个包含你的文本的文档放到浏览器中，它就会大声读给你听。如果您想停下来修复错误，可以暂停播放。

这个特定的阅读器存在一些问题，可能会因程序而异。首先，它对首字母缩略词有点困惑，并试图将它们作为一个单词来发音。好几次，它读一个句子的第一个大写字母，然后完成这个单词，就好像他们是两个独立的想法。我的建议是在你真正需要它之前测试一个阅读器，这样你就能理解它的怪癖；然后一定要阅读你要检查的任何文件，还有 T2。

最后，大声朗读你的作品也是一件好事。它帮助你理解你所说的话的流程，这是机器人声音无法做到的——但是，当你写“and”而不是“an”时，机器人声音会注意到，这是一个非常有用的工具。
