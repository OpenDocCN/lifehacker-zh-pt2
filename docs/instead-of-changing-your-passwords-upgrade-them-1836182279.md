# 升级密码，而不是更改密码

> 原文:[https://life hacker . com/与其更改您的密码，不如升级它们-1836182279](https://lifehacker.com/instead-of-changing-your-passwords-upgrade-them-1836182279)

是什么让 [成为伟大的密码](https://lifehacker.com/how-to-create-secure-passwords-that-arent-impossible-to-1825048324) ？长的东西，用 [随机化的东西，各种字符](https://lifehacker.com/how-password-constraints-give-you-a-false-sense-of-secu-1830564360)——数字、符号和字母的所有情况——以及，理想情况下，用 [备份的东西，一种二次认证方法](https://lifehacker.com/two-factor-authentication-isnt-enough-to-keep-your-acco-1827867557) 。换句话说，*你*越容易记住你的密码，别人就越容易破解(一般来说)。

虽然这应该是任何拥有 [的人都必须创建密码](https://lifehacker.com/donald-was-one-of-the-worst-passwords-you-used-in-201-1831046351) 的常识，并且有 [许多工具](https://lifehacker.com/the-five-best-password-managers-5529133) 可以用来生成、存储和调用伟大的密码，但是对于这个过程有一个小小的警告，你可能没有想过太多。你应该多久*更改*一次你的密码？

你可能在工作中最常遇到这种情况——一些烦人的通知或电子邮件让你知道是时候(再次)更改你的密码了。这可能是一个繁琐的过程，尤其是如果你必须跨多个应用程序和设备更新密码。

事实证明，这整个过程是非常不必要的。只要你拥有以 开头的 [强密码，它的存在并不会让*变得不那么*强。在一篇详述微软为何从 Windows 10 和 Windows Server 2019 的基线安全设置中取消密码到期策略的博客文章中，微软“Windows 呆子”和安全专家 Aaron Margosis](https://lifehacker.com/password-formulas-don-t-fool-hackers-1826238163) [写道](https://blogs.technet.microsoft.com/secguide/2019/05/23/security-baseline-final-for-windows-10-v1903-and-windows-server-v1903/) :

> *“定期密码到期仅仅是对密码(或散列)在其有效期内被窃取并被未授权实体使用的可能性的防御。如果密码从未被盗，就没有必要让它过期。如果你有证据表明密码被盗，你可能会立即采取行动，而不是等到过期再去解决问题。*

> 如果已知密码可能被盗，那么允许窃贼继续使用被盗密码的可接受时间长度是多少？Windows 默认为 42 天。那不觉得时间长得可笑吗？是的，但我们目前的基线是 60 天——过去是 90 天——因为强制频繁到期会带来自身的问题。如果密码不一定会被窃取，那么你就获得了这些问题，却没有任何好处。此外，如果您的用户是那种愿意在停车场回答调查，用糖果换取密码的人，任何密码过期政策都不会帮助您。

我是一个狂热的[1 密码](https://lifehacker.com/the-beginners-guide-to-1password-1794464866) 用户——爱死它了——我很欣赏这个应用程序如何竭尽全力让你知道你正在使用的密码何时可能不安全或被泄露。按照微软的建议，它*不会给你带来任何痛苦，因为你使用的密码是 x 天前(或 x 年前)的。*

也就是说，有一个*有价值的理由来改变你的密码——不管这是一个强迫的过程还是你自己决定的过程。如果你是那种不会 [检查](https://lifehacker.com/make-sure-your-passwords-stay-up-to-date-with-this-goog-1832359833) 如果你使用的 [密码已经被泄露](https://haveibeenpwned.com/) 的人，那么定期想出新密码至少是一个很好的应对可能公开的较弱密码的一揽子措施。*

对此，我提供一个替代建议:与其按照的任意时间表*更改*你的密码，你应该*升级*你的密码。如果你是一个完美的密码创建者，你可能不需要这一步。但是，如果你像我一样是普通人，你有时会为你正在尝试的新服务使用较弱的密码，因为你不想被打开密码管理器和召唤 22 个字符的怪物所困扰，你应该安排时间检查并升级你的较弱密码到更安全的密码。

如果你正在使用密码管理器，这是非常容易做到的，因为你可以扫描保存的密码列表，并开始更新任何不寻常的密码:“cat12345”，而不是“1Jf*@4，f@a9！04#*5vka*4&5%。”不过，你应该*也*已经有了一个很好的主意，你是否正在为你最喜欢的应用和服务使用弱密码——如果你根本没有使用任何密码管理器，这种可能性甚至更大。

如果您有大量的弱密码，这将是一个冗长乏味的过程，但是您总是可以战略性地思考。从你最常用的账户开始，然后一步步往下。(同样，一个密码管理应用程序会让这个过程变得简单，一个好的应用程序会告诉你什么时候你在使用一个较弱的服务密码。)

当然，即使是最强大的密码也会受益于增强:尽可能使用多因素身份验证，你的帐户将会更加安全。然后打印这篇文章——或者微软的博客文章——当你今年第八次被迫更改密码时，把它交给你的 it 团队。
