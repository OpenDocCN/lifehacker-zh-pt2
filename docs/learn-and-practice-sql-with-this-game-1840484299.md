# 用这个游戏学习和练习 SQL

> 原文:[https://life hacker . com/learn-and-practice-SQL-with the-game-1840484299](https://lifehacker.com/learn-and-practice-sql-with-this-game-1840484299)

有没有尝试过学习 SQL，这种查询语言可以让你深入了解数据库的内部？大多数教程首先让您创建自己的数据库，用无意义的内容填充它，然后运行会让您眼花缭乱的查询(万岁，我们可以模拟小部件工厂的会计部门)。骑士实验室的 [SQL 谋杀 Myster](https://mystery.knightlab.com/) y 然而，让事情变得有趣了一点。

在游戏中，你知道发生了一起谋杀案，但你丢失了描述该案的警方报告。只知道日期和地点，你能在警方数据库中筛选信息，找出凶手是谁吗？

这个场景有点做作，但它像一个谜题，因为你有一个问题迫使你去钻研数据。我的 SQL 有点生疏了，所以我立即谷歌了一下我已经忘记的语法，然后开始钻研。毕竟，这就是我们在现实生活中解决编码问题的方式:弄清楚你需要什么工具来回答你真正关心的问题。

在游戏中，你首先要找到自己的方向——这个数据库中有哪些表？—当你这么做的时候，你会找到线索去跟进。比如，警方报告很容易找到一个条目，却没有提到嫌疑人；相反，它只提到两个证人。在另一个表格中，你可以找到证人访谈，然后追踪线索(例如，一个证人记得罪犯的部分车牌)。)

我以前使用过 SQL 数据库，起初我对数据库的设置方式很恼火。我会用不同的方式来做这件事。)然而，在现实生活中，你经常不得不从一个组织混乱的数据库中提取信息。也没有内部解释为什么警察数据库也包含有用但不相关的信息，如当地健身房的会员签到。但是，虽然在现实世界中在同一个数据库中遇到所有这些事情可能没有意义，但它确实让您了解了一条信息(车牌、在健身房的一个镜头)如何能够让您访问整个进一步的信息宝库，如果您可以找到正确的数据源的话。

如果您已经精通 SQL，这个游戏可能会花费您 10-15 分钟，或者如果您还在学习，您可以使用它来指导一个更长的会话。(有一个 [伴随演练](https://mystery.knightlab.com/walkthrough.html) 边走边教。)最后的奖励等级挑战你找出是谁雇佣了杀手，并通过三张桌子的连接来做到这一点(或者至少我是这样做的)。因为如果你真的想学习如何使用一个工具，你需要弄清楚如何问和回答你自己的问题，而不是仅仅浏览一本教科书。
